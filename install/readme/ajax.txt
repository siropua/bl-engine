Обращение такое:

site.ru/json/$MODULE/* - ПРЕДПОЧТИТЕЛЬНОЕ, результат будет ВСЕГДА в формате JSON, даже если метод вернёт NULL
site.ru/ajax/$MODULE/* - Результат будет в JSON если в объекте/методе не указать обратное


Процесс:
Всё обслуживается файлом /ajax.php

Вначале выбираем, какой файл подключать в качестве обработчика:

1. Ищем site/modules/$MODULE/ajax.php, если найден, проверяем есть ли там класс rMyModuleAJAX и метод ::RunAJAX().
2. site/modules/$MODULE/$MODULE.php, есть ли в классе rMyModule метод ::RunAJAX()
3. site/modules/$MODULE.php, если в классе rMyModule метод RunAJAX()
4. (DEPRECATED!) Ищем site/modules-ajax/$MODULE.php и в нём класс rMyModuleAJAX и метод ::RunAJAX()


Варианты ответа:
*** JSON:
   array(
   	'status' => [200/404] // В общем, если 200 - всё ок. Остальные коды - ошибочные коды стандарта HTTP
   	'error_msg' => Строка ошибки, если была ошибка. Если ошибки нет - пустая строка (для совместимости)
   	'data' => Данные, которые вернул метод ::RunAJAX()
   )
*** RAW AJAX:
Тупо текстом всякая байда. 

В случае, если мы обращаемся к /ajax/ то результат будет закодирован в JSON только если метод вернёт не NULL, во всех остальных случаях - ошибки, 404, итд итп - будет возвращен текст.

В случае, если мы обращаемся к site.ru/json/, то результат всегда будет закодирован в JSON (однако, если внутри метода вызвать ::render())
